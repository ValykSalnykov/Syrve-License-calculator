# Описание проекта: Syrve License Calculator

## Назначение
Одностраничное веб‑приложение для расчёта стоимости лицензий Syrve (подписка Cloud/LT, планы PRO/Enterprise), дозакупки POS‑фронтов и дополнительных модулей. Пользователь подбирает конфигурацию, видит итоговые ежемесячные и разовые платежи и может скопировать готовый текст для согласования с клиентом.

## Технологии и структура
- **Стек:** React + TypeScript, сборка через Vite.
- **Главный экран:** `App.tsx` — управляет состоянием, расчётами и разметкой.
- **Компоненты:** `components/LicenseRow.tsx` — единый ряд выбора количества для любого дополнительного модуля.
- **Данные и типы:**  
  - `constants.ts` — прайс‑таблица фронтов, справочник POS‑лицензий и список всех дополнительных лицензий/интеграций.  
  - `types.ts` — перечисления планов и типов размещения, описания сущностей (FrontPricingTier, FrontLicenseInfo, AddonLicense).
- **Точка входа / сборка:** `index.tsx`, `index.html`, `vite.config.ts`, `tsconfig.json`.

## Основные функции
1. **Настройки размещения и плана**  
   - Переключатель Cloud ↔ LT (`DeploymentType`). При выборе LT автоматически сбрасываются дозамовленные фронты.  
   - Переключатель плана PRO ↔ Enterprise (`PlanType`). Блокируется в LT, так как расчёт подписки недоступен.
2. **Учет текущих фронтов**  
   - Выпадающий список 1–50 «Поточні фронти» задаёт исходное количество POS‑терминалов для расчёта новых фронтов.
3. **Дозамовление POS (только Cloud)**  
   - Счётчик количества новых фронтов. В LT выводится предупреждение и кнопки отключены.  
   - Аккордеон «Детальна інформація про ліцензії POS» показывает статусы поставки для Cloud/LT из `FRONT_LICENSES_LIST`.
4. **Дополнительные лицензии**  
   - Секции «Мобільні додатки» и «Інші ліцензії та інтеграції». Деление основано на списке `ADDON_LICENSES` с фильтрацией по `MOBILE_IDS` (определён в `App.tsx`).  
   - Каждый пункт: название, оригинальное имя, краткое описание/примечания, статус поставки (включено или требуется доплата) и счётчик количества.
5. **Расчёт стоимости**  
   - **Фронты (Cloud):** для каждого нового фронта выбирается ценовой уровень из `FRONT_PRICING_TABLE` (PRO/Enterprise) с суммированием ежемесячной платы и внедрения.  
   - **Add‑ons:** для каждого выбранного модуля рассчитываются ежемесячные и разовые суммы (`monthlyUah`, `setupUsd`, признак «від»/бесплатно). В LT рядом с суммой выводится пометка, что используется Cloud‑цена.  
   - Итоги выводятся в блоке «Разом до сплати»: ежемесячный платёж (грн) и разовое внедрение (USD), плюс детализация позиций.
6. **Готовый текст для клиента**  
   - Формируется сообщение с суммарными цифрами и списком деталей. Кнопка «Копіювати» записывает текст в буфер обмена и кратко подтверждает копирование.
7. **Адаптивность**  
   - Основная сетка: 4 колонки на широких экранах (настройки/фронты, мобильные модули, другие лицензии, результаты).  
   - На мобильных устройствах есть нижняя панель с итогами и кнопкой прокрутки к тексту ответа.

## Взаимосвязи и зависимости
- **Тип размещения влияет на расчёт и UI:**  
  - LT блокирует выбор плана и дозамовление фронтов; в расчёте фронтов появляется предупреждение, а для add‑ons показывается пометка о Cloud‑цене.  
  - Cloud включает расчёт дополнительных фронтов по `FRONT_PRICING_TABLE`.
- **План влияет на стоимость фронтов:** выбирается колонка PRO или Enterprise при подсчёте новых POS.
- **Выбор количества модулей:** каждый инкремент/декремент обновляет агрегированную сумму и детализацию.

## Как расширять
- **Новые модули или цены:** добавлять/редактировать элементы в `ADDON_LICENSES` и `FRONT_PRICING_TABLE` (можно задавать пометки `requiresEnterprise`, `setupIsFrom`, `note` и флаги включённости).  
- **Логика отображения:** поведение по Cloud/LT и PRO/Enterprise определяется в `App.tsx` (хук `useEffect` для сброса фронтов, `useMemo` для расчётов).
